# 계약서 비교 (로컬 전용)

브라우저에서만 `.docx` 또는 `.pdf` 두 파일을 읽어 좌우로 미리보기하고, 문자 단위 차이를 **초록색**으로 표시합니다. **원문은 네트워크로 전송되지 않습니다** (순수 프론트엔드 정적 앱).

## 요구 사항

- [Node.js](https://nodejs.org/) 20 이상 (로컬 빌드·실행용)
- npm (Node 설치 시 포함)

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 표시되는 주소(기본 `http://localhost:5173`)로 접속합니다.

프로덕션 빌드:

```bash
npm run build
npm run preview
```

`dist` 폴더만 사내 웹 서버에 올려도 동작합니다(정적 파일). **백엔드 API는 없습니다.**

## 사용 방법

1. 왼쪽·오른쪽에 각각 같은 형식의 파일을 선택합니다 (`.docx`–`.docx` 또는 `.pdf`–`.pdf`).
2. **분석**을 누릅니다.
3. 아래에 변환·렌더된 미리보기가 나타나고, 달라진 구간이 초록색으로 강조됩니다.

## 형식별 동작

- **Word (.docx)**  
  [mammoth](https://github.com/mwilliamson/mammoth.js)로 HTML로 변환한 뒤 표시합니다. Word 화면과 픽셀 단위로 같지는 않을 수 있습니다. 차이는 `textContent` 기준 문자열에 대해 [diff-match-patch](https://github.com/google/diff-match-patch)로 계산하고, DOM 텍스트 노드에 `<mark>`를 씌웁니다.

- **PDF**  
  [pdf.js](https://mozilla.github.io/pdf.js/)로 페이지를 이미지로 렌더하고, 같은 페이지의 추출 텍스트끼리 비교해 바운딩 박스에 반투명 초록 오버레이를 올립니다. 복잡한 레이아웃·스캔 PDF에서는 추출 순서·박스 위치가 어긋날 수 있습니다.

## 보안·DLP 관련

- 이 저장소의 코드는 **사용자가 고른 파일을 어디로도 `fetch`하지 않습니다.** 처리는 전부 브라우저 메모리 안에서 이루어집니다.
- 회사 PC의 DLP가 “로컬 파일 읽기”나 “로컬 개발 서버” 자체를 막는 경우는 제품마다 달라, 여기서는 보장할 수 없습니다. 그런 환경에서는 오프라인 PC나 허용된 정적 호스팅만 사용하는 방식을 검토하세요.

## 제한 사항

- 암호가 걸린 PDF·손상된 파일은 열리지 않을 수 있습니다.
- `.doc`(구형 바이너리 Word)은 지원하지 않습니다.
- 좌우 **형식 혼합**(docx vs pdf) 비교는 지원하지 않습니다.
